# Vue 两个版本

  • Runtime Only（只包含运行时：需预编译代码）
    不带编译模块。无法编译 template 属性， 当 vue 实例中出现 template 会报错！因此需要借助 wepack 的 vue-loader 工具把 .vue 文件编译成 JavaScript ，因为是在编译阶段做的，所以它只包含运行时的代码！该代码体积也会更轻量。
// 需要编译器
new Vue({
  template: '<div>{{ hi }}</div>'
})
  • Runtime Compiler（运行时 + 编译器）
    包含预编译模块。Vue 最终渲染是通过 render 函数，如果写 template 属性，则需要编译成 render 函数，那么这个编译过程会发生运行时，所以需要带有编译器的版本。该编译过程对性能有一定损耗!
//不需要编译器，下面是vue-cli 初始化的 vue 项目
import Vue from 'vue'
import App from './App.vue'
import router from './router'
import store from './store'

console.log(App);

Vue.config.productionTip = false

new Vue({
  router,
  store,
  render: h => h(App)
}).$mount('#app')

下面写法会报错:

import Vue from 'vue'
import App from './App.vue'
import router from './router'
import store from './store'

console.log(App);

Vue.config.productionTip = false

new Vue({
  router,
  store,
  template: "<h1>hell world</h1>"   //vue-cli 默认使用runtime-only版，此处会报错
  // render: h => h(App)
}).$mount('#app')

  • 模板编译
    Vue 编译模块会将 template 模板编译成 render 函数
